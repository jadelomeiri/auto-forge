# Forge — Product

## One-line description

Forge is a convention-first full-stack framework for building readable business apps fast.

## Short pitch

Forge brings back the joy of Grails for 2026:
- convention over configuration
- batteries included
- server-rendered HTML by default
- REST APIs by default
- readable generated code
- AI-friendly project structure
- explainable conventions

Forge is designed for developers who want momentum, clarity, and a coherent full-stack app story.

## Core promise

A developer should be able to:
1. create a new app
2. generate a model
3. generate CRUD scaffolding
4. run migrations
5. start the app
6. have a usable business app in minutes

## Who Forge is for

- internal tools
- admin dashboards
- CRUD-heavy business apps
- operational tools
- monolithic apps
- server-rendered apps
- simple APIs

## Who Forge is NOT for

- SPA-first frameworks
- microservice platforms
- infra frameworks
- prompt-to-app tools
- plugin ecosystems (v1)

## Principles

1. Convention is the API
2. Generated code must stay readable
3. Monolith first
4. HTML first, API also first-class
5. AI-friendly, not AI-dependent
6. Explainability over magic

## AI-native meaning

- machine-readable manifest
- deterministic generators
- explain commands
- simple project structure
- predictable conventions

## v1 scope

- app generator
- model generator
- scaffold generator
- migrations
- server-rendered views
- validation
- tests
- manifest
- explain commands

## Non goals

- plugin system
- realtime
- SPA framework
- custom ORM
- prompt-based generation
- microservices

## Emotional goal

Forge should feel:

- calm
- readable
- opinionated
- productive
- trustworthy
- coherent

Desired reaction:

"This feels simple. This feels fast. This feels right."
